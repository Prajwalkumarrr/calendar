Quickstart

Make your first API call in 60 seconds. Get a personal access token in Settings → Developers, then:

# Fetch your next 5 events curl -H "Authorization: Bearer evt_pat_..." \ https://api.elevaite.so/v1/events?limit=5 # Response { "data": [ { "id": "e_01H8K3M...", "title": "Roadmap review · Q3", "start": "2026-05-20T11:30:00-07:00", "end": "2026-05-20T12:30:00-07:00", "calendar_id": "c_work" } ] }

Authentication

Two flavors:

  • Bearer evt_pat_…Personal access tokens — for scripts and your own automations.
  • OAuth 2.0For apps acting on behalf of other ElevAIte users. PKCE supported.

Core endpoints

Everything follows REST. JSON in, JSON out. No XML, ever.

GET/v1/events
List events in a time range. Supports filters: calendar, attendee, color.
POST/v1/events
Create an event. Supports recurrence (RRULE), attendees, conferencing.
GET/v1/events/:id
Fetch a single event with all metadata expanded.
DEL/v1/events/:id
Delete an event. Notifies attendees by default.
GET/v1/scheduling-links
List your scheduling links. Includes booking counts.
POST/v1/availability
Get free/busy across team members in one call.

Webhooks

Subscribe to events as they happen. We retry with exponential backoff for 24 hours.

  • event.createdFires when any event is created on a connected calendar.
  • event.updatedTime, title, attendees changed.
  • event.deletedFires when an event is removed.
  • booking.createdSomeone booked one of your scheduling links.
  • booking.cancelledInvitee cancelled.

Rate limits

Free & Personal: 60 req/min · Team: 300 req/min · Enterprise: custom. Headers X-RateLimit-Remaining and X-RateLimit-Reset tell you where you stand.
